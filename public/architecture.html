Hidden App — Architecture Diagrams

Expedition-tech platform for remote tourism in Colombia · PWA, Firebase/GCP, multi-agent AI ecosystem scoped by department

Production PWA Hackathon demo video Source code Architecture (Markdown)

Client (React 19 PWA + Capacitor), Firebase/GCP backend, ADK runtime with persistent sessions, the multi-agent expedition pipeline, MCP catalog server, Cloud Run Live voice worker and external telemetry APIs.

One chat turn: the hyperlocal agent loads its persistent ADK session (Firestore), queries the catalog on demand (MCP / FunctionTools, hermetically scoped to one departmentId), can invoke the Ranger as a tool (getLiveConditions) and checks real routes before answering with validated widgets.

"Plan my 3-day expedition" → the chat enqueues an expedition; three specialist ADK agents build the itinerary exclusively from verified catalog data with real Google Routes legs, streaming live progress (queued → curating → routing → writing → ready) to the chat widget via onSnapshot.

One monitored destination: Ranger telemetry analysis every 15 minutes while the app is open, plus cloud cron push alerts for UV, AQI and rain thresholds.

Text agents (ADK)
chatAgent and environmentalAgent run on Google ADK — Agentic RAG, MCP catalog tools, persistent Firestore sessions, structured JSON.
Multi-agent expedition planner
planExpedition chat tool + onExpeditionCreate trigger: curator → logistics → writer LlmAgents, deterministic data gathering, validated ids, real driving legs.
Ranger as a tool
getLiveConditions fetches live weather/AQI/marine telemetry and delegates the tactical analysis to the Ranger sub-agent inside the chat turn.
Live voice
Cloud Run hidden-agent-worker + LiveKit + Gemini Multimodal Live (separate from the ADK text stack).
Off-Grid Vault
Department SQLite packs + local RAG on device — search and offline chat with zero network.
